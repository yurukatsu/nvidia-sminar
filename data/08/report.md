## 全体まとめ

Stockmark CTO の有馬氏が、日本の製造業で使われる複雑かつ機密性の高い技術文書を読み解くための **ソブリン AI / ドキュメント読解特化 VLM** の開発について講演。

日本のビジネス文書は「漫画やアニメと同レベルにクリエイティブ」で、表・図面・フローチャート・手書き化学式等を含むビジュアルリッチなドキュメントが多く、汎用 LLM では読解が困難。Stockmark はフルスクラッチで **1,000億パラメータ LLM**（StockMark-2）を開発した後、さらに **マルチモーダル VLM** を構築。日本語の複雑ドキュメント向け合成教師データを NVIDIA の技術（NeMo RL、TensorRT LLM、**Nemotron Personas Japan**）を活用して大量生成し、フロンティアモデルをも上回るドキュメント読解精度を実現した。

実応用として、製造業の技術ナレッジグラフ構築→社会課題とのマッチング→ビジネスアイデア自動生成、さらに Nemotron Personas Japan による市場性評価シミュレーションまでの一連のパイプラインを紹介。

## Key Points

### 1. Stockmark の概要とプロダクト

- 2016年創業、東京、従業員約150名、エンタープライズ300社超（7割が製造業）
- **A-Connect**：製造業技術者向け情報収集 SaaS。技術者の仕事の約65%が情報収集（ニュース・特許・論文・社内データ）に費やされている現状を AI で支援
- **SAT**：エンタープライズの複雑文書のデータ構造化・クレンジング SaaS

### 2. StockMark-2 — 1,000億パラメータ LLM のフルスクラッチ開発

- **規模**：1,000億パラメータ、日本語重点
- **インフラ**：NVIDIA H100 Tensor Core GPU × 320枚（40ノード × 8枚）、約6ヶ月間の学習
- **並列計算**：NVIDIA Megatron-LM を使用
- **性能**：MT-Bench で Llama 3.3 等のグローバルモデルや国内フルスクラッチモデルの中で最高性能クラス（2025年当時）
- **特長**：日本のビジネス話題に強い、ハルシネーションが低い
- **NVIDIA NIM** として利用可能。推論速度が従来比最大2.5倍に高速化。プライベートクラウド/オンプレミスでの機密環境デプロイが可能

### 3. マルチモーダル VLM の開発

- **規模**：10億パラメータのマルチモーダルモデル
- 表・チャート・スライド等のビジュアルドキュメントの QA ベンチマークで GPT-4 等のフロンティアモデルを上回る精度
- **学習データ**：独自収集した数万件規模のビジネスドキュメントに対し、人間がどのように読み解くかの正解データを人力で作成
- 「人間がこう読み解く」という理解過程を学習させることで、複雑な資料の人間らしい読解能力を獲得

### 4. NVIDIA 技術の活用

| 技術 | 用途 |
| --- | --- |
| **NeMo RL**（旧 NeMo Aligner） | SFT、DPO（人間の好みの学習）、GRPO による強化学習 |
| **TensorRT LLM** | 合成教師データの高速生成 |
| **Morpheus Retriever** | RAG 実現のためのエンベディング |
| **Nemotron Personas Japan** | 多様なビジュアルリッチドキュメントの合成データ生成のシード（600万ペルソナ） |

### 5. 合成教師データ生成パイプライン

#### パイプライン1：英語データセットの日本語化

```
英語画像 + 英語 QA ペア
    ↓
Qwen 等の OSS モデルに入力
    ↓
日本語 QA ペア + 思考過程（Chain of Thought）を生成
    ↓
品質フィルタリング（GPT/Qwen 等でダブルチェック、整合性の低いものを排除）
    ↓
合成教師データセット（画像 + 日本語 QA + 思考過程）
```

#### パイプライン2：ドキュメント系合成データ

```
表・スライド等のドキュメント
    ↓
Structure-Level Annotation（人力で文書構造を文字起こし：ヘッダー、タイトル、テキスト、フッター等）
    ↓
構造化データに対して Qwen 等で日本語 QA ペア + 思考過程を生成
    ↓
品質フィルタリング（整合性の低いものを排除）
    ↓
合成教師データセット（ドキュメント + 日本語 QA + 思考過程）
```

#### パイプライン3：Nemotron Personas Japan を活用したビジュアルリッチドキュメント生成（7ステップ）

```
Step 1: Topic Generation — Nemotron Personas Japan（600万ペルソナ）をシードに
         ペルソナベースのドキュメントトピック生成（小売業、会計、看護等の多様な業界）
Step 2: Metadata Generation — 業務ドキュメントの XML 的構造を自動生成
Step 3: Image Generation — メタデータから画像をレンダリング（フローチャート、設計書等）
Step 4: Description Generation — 画像の詳細な説明を自動生成
Step 5: QA Pair Generation — ドキュメントに対する質問・回答・思考過程を自動生成
Step 6: 各業界トピックに対して Step 2-5 を大量実行
Step 7: 整合性フィルタリング（低品質データを排除）
```

→ 多様な業界のビジュアルリッチドキュメント（フローチャート、設計書、化学式等）の教師データを大量生成

### 6. 成果 — フロンティアモデルを上回る読解精度

- **手書き化学式**の読解が可能に（GPT-5、Gemini 3 でも困難なタスク）
- **機械系図面**の寸法読み取りが可能に
- 専門的文書（化学式、図面、複雑な経済レポート）で高精度を達成しつつ、一般的な日本語ドキュメント読解でも OSS / フロンティアモデルと同等以上の性能を維持（チェリーピッキングではない）

### 7. 導入事例

| 企業 | 活用内容 |
| --- | --- |
| **パナソニック** | 機密性の高い業務文書に対し追加事前学習を実施。業務知識を理解させ各事業セグメントに適応 |
| **日本製鋼所** | 用途探索：社内技術文書と公開情報を読解し、自社技術が世の中のどの課題に対応できるかを AI が自動でアイデア生成 |
| **産総研（AIST）** | 産学連携ビジネスマッチング：研究者のプロフィール情報を VLM で読み込み、顧客ニーズに最適な研究者チームを自動提案 |

### 8. ビジネスアイデア自動生成パイプライン

```
技術文書（機密・複雑）
    ↓ VLM で読解
技術シーズのナレッジグラフ構築
    ↓
社会課題（ニュース、政府レポート、論文）
    ↓ マッチング
ビジネスアイデア自動生成
    ↓
Nemotron Personas Japan で市場性評価（600万人アンケートシミュレーション）
```

**具体例**：
- 同様期アサヒビール：立ち泡技術 → 家庭で生ビール体験の実現へ接続（本来泡技術はビール用途で開発されたわけではない）
- 日本製鋼所：セラミック技術 → どの市場で活用可能かを自動探索

### 9. Nemotron Personas Japan による市場評価シミュレーション

- 600万ペルソナに対しアンケートをシミュレーション
- **検証例**：キャッシュレス決済の利用率調査
  - 実際の人間による調査結果と Stockmark モデルの出力が近似する分布を実現
  - GPT では再現できなかった分布も Stockmark モデルでは近似に成功
- 定性的なヒアリング（「なぜ使わないのか」等）も人間らしい回答を取得可能
- → ビジネスアイデアの市場性評価を自動化

## 当チームへの示唆

- **ドキュメント読解特化 VLM の可能性**：金融業界でも表・チャート・図面を含む複雑な文書（有価証券報告書、運用報告書、リサーチレポート等）の読解ニーズは高い。Stockmark の VLM 開発アプローチ（人間の読解過程の学習 + 合成データ大量生成）は当チームの文書処理自動化に参考になる
- **合成教師データ生成パイプライン**：日本語ドキュメントの学習データ不足を NVIDIA 技術で克服した手法は、当チームが金融ドメイン特化の VLM/LLM を構築する際にも直接応用可能
- **Nemotron Personas Japan の活用**：600万ペルソナによるアンケートシミュレーションは、市場調査・顧客分析・投資アイデア評価に新しいアプローチを提供。人間調査との近似が確認されたことは信頼性の根拠になる
- **技術シーズ × 社会課題マッチング**：ナレッジグラフ構築→ビジネスアイデア生成のパイプラインは、投資先の技術資産評価や新規ビジネス機会の探索に応用可能なフレームワーク
- **ソブリン AI / NIM のデプロイモデル**：機密環境でのオンプレミスデプロイが可能な NIM 化は、金融機関の機密性要件を満たす LLM 運用の選択肢として検討に値する
