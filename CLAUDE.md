# CLAUDE

## 目的

イベントの参加報告書を所属チーム向けに作成します。
チームではナレッジを Obsidian で管理しているあめ、必ず **Obsidian に対応した** Markdown ファイルを作成してください。

## データのフォルダ構成

- `transcript.json`: 講演のトランスクリプトの書き起こしデータが入っています
- `info.md`: 講演の基本情報が記載されています
- `slide.*`: 講演で使用されたスライドです。（ない場合もあります）

米 これらのファイルが格納されたディレクトリを指定します

## チームについて

私はマーケッツテクノロジー部データサイエンスチームに所属しています。
当部はバンキング運用部署、ALM部署、セールス&トレーディング部署に向けて、市場データ分析、投資戦略開発、業務の効率化をしています。
生成AI、Ai Agents活用にも積極的です。

## 内容

まずは講演の全体的なまとめてください。
まとめる際は、チームの業務に役立ちそうなポイントを中心にまとめ、そのまま報告しやすい形をとってください。

### 構成

最低限、以下の内容を含んだ Markdown を作成してください

- プロパティ
- 会議情報
- 全体的なまとめ
- Key Points

### プロパティ

プロパティには、

- title
- author
- created
- tags

を利用してください。

タグに関しては、5個程度作成してください。

## 出力

markdown の出力は、インプットデータと同じディレクトリ内に、`report.md` という名前で保存してください
